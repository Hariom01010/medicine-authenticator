# Project Knowledge Transfer: Medicine Authenticator

## 1. Project Overview
The **Medicine Authenticator** is a proof-of-concept application designed to provide transparency and security in the pharmaceutical supply chain. It leverages a simplified, custom-built **Blockchain** to create an immutable ledger for each medicine batch, allowing stakeholders to verify the authenticity and history of products.

## 2. Core Problem Solved
Counterfeit medicines are a global health crisis. Traditional tracking systems can be tampered with in centralized databases. This project demonstrates how **decentralized, cryptographic linking** can make unauthorized data modification immediately visible to anyone in the chain.

## 3. Technology Stack
- **Frontend**: React.js (Vite)
- **Backend**: Node.js, Express.js
- **Database**: MongoDB (via `mongoose` and `mongodb-memory-server` for dev)
- **Security**: SHA-256 Hashing (via Node's `crypto` module)

## 4. Architecture & Logic

### Simplified Blockchain Implementation
The core logic resides in `backend/services/blockchain.js`. 
- **Block**: Each "event" (Creation, Shipping, Receiving) is a block.
- **Linking**: Every block contains the `hash` of the previous block (`previousHash`).
- **Immutability**: Since the current block's hash is derived from the `previousHash`, changing any data in an earlier block breaks the entire chain's cryptographic validity.

### Data Model (`backend/models/Batch.js`)
- **Batch**: Represents a specific production run of a medicine.
- **Chain**: An array of `Block` objects stored within the Batch document.

## 5. Key Features

### Create Batch
Initializes a new medicine record and creates the **Genesis Block**. This block captures the manufacturer and production start date.

### Add Updates (Update Batch)
Allows adding a new "hop" in the supply chain (e.g., "Shipped to Warehouse A"). This adds a new block to the batch's chain, linked to the last known state.

### Verification (`/verify` endpoint)
The system performs a "Full Chain Audit":
1. Recalculates the hash for every block starting from the first.
2. Checks if the `previousHash` property matches the actual hash of the preceding block.
3. Returns `isValid: true` only if the integrity is perfect.

### Tamper Simulation
Specifically included for demo purposes. It maliciously modifies the database data without updating hashes, demonstrating how the verification logic detects the "Silent Hack."
